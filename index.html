🔗 Subscribe (auto-updating)
Add a subscription URL to your calendar app. Your phone or computer's calendar will check for updates periodically (typically every few hours) and pull in any schedule changes automatically — no need to re-download.
How to use it:
  • iPhone / Apple Calendar: Settings → Calendar → Accounts → Add Account → Other → Add Subscribed Calendar → paste URL
  • Google Calendar: calendar.google.com → "Other calendars" + → "From URL" → paste
  • Outlook: Calendar → Add calendar → Subscribe from web → paste URL
⚠ Privacy: This URL contains a token that lets anyone view your schedule. Don't share it. To revoke access (if leaked), click "Generate URL" again — old URL stops working.

▶ One-time setup: deploy the create-user edge function
To create physician accounts directly (so they can log in without self-registering), you need a Supabase Edge Function with admin privileges. This function runs with the service role key so it can create auth users on your behalf, and verifies the caller is an admin before doing so.
⚠ Security: This function uses your service role key, which bypasses Row-Level Security. The function verifies the caller is an admin by checking Supabase Auth user metadata (role: 'admin') before performing any admin action. Never expose the service role key in the browser — it only lives as a secret on the edge function.
  1. In your Supabase dashboard → Edge Functions → Create function → name it create-user.
  2. Paste the Deno code below into the function editor.
  3. In the function's Secrets settings, add:
    • SUPABASE_SERVICE_ROLE_KEY = your project's service role key
      (Found in Project Settings → API → service_role secret — the one marked "secret")
  4. In function settings, ensure verify_jwt is enabled (default).
  5. Deploy the function.
  6. Make sure your own admin account in Supabase Auth has role: 'admin' in its user_metadata. If you're not sure, expand the "Bootstrap first admin" section below after deploying.

▶ Bootstrap first admin — grant yourself admin role in Supabase Auth
If you're seeing HTTP 403: Caller is not an admin when creating users, your Supabase Auth user is missing the role: 'admin' tag in its user_metadata. The app's local USERS array says you're an admin, but Supabase Auth doesn't know that. Fix it in one of three ways:
Method A — Supabase dashboard (easiest, ~30 seconds):
  1. Go to Supabase dashboard → Authentication → Users.
  2. Find your admin email, click the row.
  3. Scroll down to User Metadata (or "Raw User Meta Data").
  4. Edit the JSON to include "role": "admin". Example: 
    {
  "role": "admin",
  "first": "Your",
  "last": "Name"
}
  5. Save. Then sign out of RadScheduler and sign back in so your new token carries the admin role.
Method B — SQL editor (if you have many admins): 
-- Run in Supabase → SQL Editor. Replace the email first.
UPDATE auth.users
SET raw_user_meta_data = raw_user_meta_data || '{"role":"admin"}'::jsonb
WHERE email = 'your-admin@example.com';
Then sign out and sign back in.
Method C — Temporary open-bootstrap endpoint: Modify the create-user function to skip the admin check on a one-time call, promote yourself, then restore the check. Only use this if Methods A and B aren't available to you. Not recommended for production deployments.
